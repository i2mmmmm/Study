1. 모듈러 연산 (나머지 연산)

   - 컴퓨터 메모리 한계 때문에 모듈러 연산으로 가두어 계산  
   - 계산 결과만 보고 판단 -> 일방향성(암호화) 가능
 
2. 최대 공약수, 유클리드 호제법 : 역수찾기 무기

   - 공개키 암호(RSA)에서 나눗셈 많이 씀 -> 모듈러는 나누기 X  
   - 곱해서 1이 되는 수 (잉여역수) modular Inverse   
     ex) modular 7 ->　　3 X 5 = 15 Ξ 1 (mod 7) =>　　3의 역수 = 5   
   - 유클리드 호제법 => 역수 눈으로 찾기 어려움 => 이걸로 최대공약수를 찾고 역수도 찾을 수 있음

3. 페르마 소정리 : 암호해독에서 중요
   공개키 암호 RSA
   - 정리 : p가 소수(Prime number) 이고, a가 p의 배수가 아닐 때, 다음 공식이 무조건 성립   
     **a^(p-1) Ξ 1 (mod p)**
   - 아무리 거대한 숫자라도 소수 P의 거듭제곱 형식으로 모듈러 연산을 하면 숫자가 1로 초기화된다는 것
   - 복잡하게 꼬아서 암호화한 거대한 숫자를 다시 원래대로 복호화하는 매커니즘 만들 수 있음
  
4. 추상대수학 : 군 Group, 환 Ring, 체 Field
   - 군 Group : 더하기(또는 곱하기) 한가지만 자유롭게 잘되는 숫자들 모임 (ex>시계 숫자들은 군)
   - 환 Ring : 더하기, 빼기, 곱하기 잘되는데 나누기는 잘 안되는 숫자 모임 (ex> 정수세계에서 3÷2 는 정수가 아님 - 환)
   - 체 Field : 더하기, 빼기, 곱하기, 나누기(0 제외) 사칙연산 다 되는 숫자 모임 (ex> 우리가 아는 실수체계)
   - 유한체(Finite Field) GF(p) / GF(2^n)  
    -> 원소개수 유한한 체 -> 컴퓨터 0,1 세계 => 유한체   
    -> AES암호화 -> 유한체 GF(2^8)위에서 쪼개고 더하고 나누면서 데이터 꼬아버림
